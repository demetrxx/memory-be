# Repository Guidelines

## Project Structure & Module Organization
This is a NestJS monorepo. The primary app lives in `apps/mono` (see
`apps/mono/src/app.module.ts`, `apps/mono/src/modules/`, `apps/mono/src/config/`,
and `apps/mono/src/common/`). Shared runtime code is in `libs/core` (e.g.
`libs/core/src/chat-queue`, `libs/core/src/db`, and `libs/core/src/index.ts`).
Build artifacts land in `dist/`. The `apps/admin` and `apps/mini-app`
directories are currently empty; add new code there only with team alignment.

## Architecture Overview
The runtime entrypoint is `apps/mono/src/main.ts`, which wires the NestJS app in
`apps/mono/src/app.module.ts`. Domain logic is organized under
`apps/mono/src/modules/`, while cross-cutting utilities and configuration live
in `apps/mono/src/common/` and `apps/mono/src/config/`. Shared infrastructure
such as database setup and queue helpers is centralized in `libs/core/src/`.

## Build, Test, and Development Commands
- `pnpm install` installs dependencies.
- `./ngrok.sh` (after `chmod +x ngrok.sh`) starts ngrok and writes `APP_URL` for
  local webhook testing.
- `pnpm run dev` runs the backend in watch mode; `pnpm run start:debug` enables
  debugger; `pnpm run start:prod` runs compiled output from `dist/`.
- `pnpm run build` compiles the apps.
- `pnpm run lint` and `pnpm run format` enforce lint and formatting rules.
- Database tooling: `pnpm run mig:generate`, `pnpm run mig:run`,
  `pnpm run mig:revert`, and `pnpm run db:drop`.

## Coding Style & Naming Conventions
TypeScript is formatted by Prettier (2-space indent, single quotes, trailing
commas). ESLint is required and enforces sorted imports (`simple-import-sort`).
Prefer the path alias `@app/core/...` for shared library imports. Tests should
use the `.spec.ts` suffix and live alongside code in `apps/` or `libs/`.

## Testing Guidelines
Jest runs unit tests via `pnpm run test` (matching `**/*.spec.ts` under
`apps/` and `libs/`). End-to-end tests live in `apps/mono/test` and run with
`pnpm run test:e2e` (config: `apps/mono/test/jest-e2e.json`). Coverage reports
are generated by `pnpm run test:cov` in `coverage/`.

## Commit & Pull Request Guidelines
Commit messages follow a Conventional Commits style seen in the history, e.g.
`feat(bot): add webhooks` or `feat(chat): v1`. PRs should include a short
description, the commands run (such as `pnpm run lint` and `pnpm run test`), and
any new or changed environment variables. Provide screenshots or logs for bot
or UI behavior changes.

## Configuration & Secrets
Use `.env` for configuration and keep secrets out of the repo. For local
development with Telegram webhooks, ensure `APP_URL` is set via `./ngrok.sh`.
